Unit 2 Data Munging [8 Hrs.]
=======================================================
    Data munging means the process of cleaning, organizing, correcting,
    and preparing raw data so that it can be used for analysis.

    Simple meaning:
        Raw data from real life is usually messy.
        Data munging makes it usable.

    Other similar terms
        Data cleaning
        Data wrangling
        Data preprocessing

    Why data munging is important
        Real-world data often contains errors
        Wrong data leads to wrong analysis
        Clean data improves model accuracy
        Good decisions depend on good quality data

    Example

        Raw student data:
            Ram       45
            Sita      78
            Hari      -
            Gita      90
            Ram       45

        Problems:
            Hari has missing mark
            Ram is repeated

        After munging:
            Remove duplicate Ram entry
            Decide how to handle Hari's missing value

        Now the data becomes more reliable.






    2.1 Data Quality
=======================================================
        Data quality means how good, accurate, complete, consistent,
        and useful the data is for a specific purpose.

        Simple meaning:
            High quality data is trustworthy and ready for use.
            Low quality data is inaccurate, incomplete, or confusing.




        Main characteristics of data quality

            Accuracy
                Data correctly represents the real-world value

                Example:
                    Student age recorded as 20
                    when actual age is 20

            Completeness
                Important values are not missing

                Example:
                    Name, age, marks, and roll number are all present

            Consistency
                The same data follows the same format and meaning everywhere

                Example:
                    Kathmandu should not appear as
                    Kathmandu, KTM, and kathmandu in the same dataset
                    unless standardized

            Validity
                Data follows rules and allowed ranges

                Example:
                    Marks should be between 0 and 100
                    Age should not be negative

            Timeliness
                Data is up to date

                Example:
                    Current stock report should reflect today's stock,
                    not last year's stock

            Uniqueness
                Same record should not appear more than once unless required

                Example:
                    One student should not be entered twice by mistake






        Why data quality matters in Data Science

            If data quality is poor:
                Analysis may be wrong
                Graphs may mislead
                Models may fail
                Decisions may be harmful






        Example of poor quality data

            Product    Price
            Rice       80
            Oil        200
            Sugar      -50

            Problem:
                Sugar price cannot be negative
                So validity is poor




        Example of good quality data

            Product    Price
            Rice       80
            Oil        200
            Sugar      120

            Now prices are meaningful and usable









        Data quality in different situations

            Education
                Correct student marks and attendance needed

            Healthcare
                Correct patient history is critical

            Banking
                Accurate transaction records are essential

            Business
                Correct sales and inventory data are needed








        How to improve data quality

            Check for missing values
            Remove duplicates
            Standardize formats
            Validate ranges
            Fix unit problems
            Use reliable data entry rules
            Regularly review and update data






        Simple maths example

            Marks:
                45, 50, 60, 70, 500

            Problem:
                500 is likely data entry error

            Mean before cleaning:
                (45 + 50 + 60 + 70 + 500) / 5
                = 725 / 5
                = 145

            This is clearly misleading.

            If corrected to 50:
                45, 50, 60, 70, 50

            Mean after correction:
                (45 + 50 + 60 + 70 + 50) / 5
                = 275 / 5
                = 55

            Lesson:
                Good data quality changes the result significantly.







    2.2 Common Data Issues
=======================================================
        Common data issues are problems that appear frequently in raw datasets.

        These issues make analysis difficult and unreliable.

        In this unit, common issues are:
            Duplicates
            Missing data
            Nonstandard data
            Unit mismatch








        Example of one messy dataset

            Name        City         Weight      Marks
            Ram         KTM          55 kg       78
            Sita        Kathmandu    121 lb      82
            Hari        Pokhara      -           65
            Ram         KTM          55 kg       78

        
                Problems:
                    Ram appears twice
                    Hari has missing weight
                    City format is inconsistent
                    Weight uses mixed units
        
                This is a typical real-world messy dataset.






    2.2.1 Duplicates
=======================================================
        Duplicates mean the same record appears more than once in a dataset.

        Simple meaning:
            One entry is repeated unnecessarily.

        Example:
            Roll    Name    Marks
            1       Ram     80
            2       Sita    75
            1       Ram     80

          Here the first and third rows are duplicates.




        Why duplicates are a problem

            They increase data size unnecessarily
            They can distort counts and averages
            They may give false impression of frequency
            They reduce trust in data





        Example with counting students

            Data:
                Ram
                Sita
                Hari
                Ram

            Count without cleaning:
                4 students

            Actual unique students:
                3 students

            So duplicate causes wrong result.






        Example with average salary

            Salaries:
                20000, 25000, 30000, 20000

            If last 20000 is a duplicate,
            then the average is affected.









        Types of duplicates

            Exact duplicates
                Entire row is repeated exactly

            Partial duplicates
                Some fields are same but not all

                Example:
                    Ram    Kathmandu    9800000000
                    Ram    Kathmandu    9800000000

                exact duplicate

                But:
                    Ram    KTM          9800000000
                    Ram    Kathmandu    9800000000

                may be duplicate person with nonstandard city name









        Causes of duplicates

            Repeated data entry
            Merging files without proper checking
            System import errors
            Typing records multiple times








        How to detect duplicates

            Check full repeated rows
            Check repeated IDs
            Check repeated name and phone combinations
            Use software tools to find repeated records








        How to handle duplicates

            Remove exact duplicates
            Review suspicious partial duplicates
            Keep only one correct record
            Use unique IDs where possible








        Example in customer data

            Customer ID    Name     City
            101            Asha     Lalitpur
            102            Bina     Bhaktapur
            101            Asha     Lalitpur

            Customer ID 101 is repeated.
            One row should likely be removed.






        Important note
            Not all repeated values are duplicates.

            Example:
                Product
                Rice
                Rice
                Sugar

            If each row means a separate sale,
            then repeated Rice may be valid.
            So duplicates must be judged in context.








    2.2.2 Missing Data
=======================================================
        Missing data means some values are not available or not recorded.

        Simple meaning:
            Some part of the dataset is empty or unknown.

        Example:
            Name      Age      Marks
            Ram       20       75
            Sita      -        82
            Hari      22       -

        Here:
            Sita's age is missing
            Hari's marks are missing





        Common ways missing data appears

            Blank cell
            NA
            Null
            Dash -
            Unknown
            Not available





        Why missing data is a problem

            Reduces completeness
            Makes analysis harder
            Can bias results
            Some algorithms cannot work with missing values
            Can reduce dataset size if rows are removed





        Causes of missing data

            Data entry skipped
            Survey not answered
            Sensor failure
            Data lost during collection
            Information not available





        Example with student marks

            Marks:
                45, 60, -, 80, 70

            If we ignore the missing value carelessly,
            average calculation may become misleading.








        Ways to handle missing data

            Remove the row
                Useful if only few rows are missing
                Risk: loss of data

            Remove the column
                Useful if too many values are missing
                Risk: loss of important feature

            Fill with mean
                For numerical data

            Fill with median
                Better when outliers exist

            Fill with mode
                For categorical data

            Fill with a fixed label
                Example:
                    Unknown

            Predict missing value
                Use model-based imputation





        Example using mean

            Data:
                10, 20, -, 40

            Mean of known values:
                (10 + 20 + 40) / 3
                = 70 / 3
                = 23.33

            Missing value may be filled with 23.33







        Example using mode

            City:
                Kathmandu
                Pokhara
                Kathmandu
                -

            Mode:
                Kathmandu

            Missing city may be filled with Kathmandu
            only if this makes practical sense








        Important caution

            Filling missing values blindly may introduce error.
            We should first ask:
                Why is the value missing?
                Is the row important?
                Is imputation reasonable?

        Example from medical data

            Patient    BP
            A          120
            B          -
            C          130

            Missing BP may be serious.
            We should not guess carelessly.

        So handling missing data depends on context.






    2.2.3 Nonstandard Data
=======================================================
        Nonstandard data means data is recorded in inconsistent formats,
        spellings, styles, or structures.

        Simple meaning:
            Same type of information is written in different ways.

        Example:
            City
            Kathmandu
            kathmandu
            KTM
            KATHMANDU

        All refer to the same city,
        but the format is inconsistent.





        Why nonstandard data is a problem

            Same category may be treated as different values
            Grouping becomes wrong
            Counting becomes wrong
            Filtering becomes difficult
            Visualization becomes misleading






        Example with city counts

            City values:
                Kathmandu
                KTM
                kathmandu
                Pokhara





            If untreated, system may count:
                Kathmandu     1
                KTM           1
                kathmandu     1
                Pokhara       1

            But actual intended count may be:
                Kathmandu     3
                Pokhara       1








        Types of nonstandard data

            Different spellings
                color, colour

            Different cases
                Apple, apple, APPLE

            Different abbreviations
                Mr., Mister

            Different date formats
                01/02/2024
                2024-02-01
                Feb 1, 2024

            Different coding styles
                Male, M, male





        Example with gender data

            Gender
            Male
            M
            male
            FEMALE
            F

            These must be standardized before analysis.








        How to handle nonstandard data

            Convert to common case
                lowercase or uppercase

            Standardize spelling
                choose one correct form

            Standardize categories
                Example:
                    M and Male both become Male

            Standardize date format
                Example:
                    YYYY-MM-DD

            Use lookup rules or dictionaries






        Example after standardization

            Before:
                KTM
                kathmandu
                Kathmandu

            After:
                Kathmandu
                Kathmandu
                Kathmandu







        Example with words

            Product review words:
                Good
                good
                GOOD

            If not standardized,
            frequency count becomes split.

            After lowercase conversion:
                good
                good
                good

            Now the word count is correct.













    2.2.4 Unit Mismatch
=======================================================
        Unit mismatch means the same type of measurement is stored
        using different units in one dataset.

        Simple meaning:
            Values look similar,
            but they are measured differently.

        Example:
            Weight
            50 kg
            120 lb
            60 kg

        Problem:
            50 and 120 cannot be compared directly
            because one is in kilograms and the other is in pounds.







        Why unit mismatch is a problem

            Causes wrong calculations
            Makes averages meaningless
            Distorts graphs
            Misleads models and analysis


        Example with weight

            Weights:
                50 kg
                120 lb

            If treated directly as:
                50 and 120

            Average becomes:
                (50 + 120) / 2 = 85

            But 85 is meaningless because units differ.




        Correct method

            Convert all values to same unit first.

            Since:
                1 kg is approximately 2.2 lb

            120 lb in kg:
                120 / 2.2
                approximately 54.5 kg

            Now weights become:
                50 kg
                54.5 kg

            Average:
                (50 + 54.5) / 2
                = 104.5 / 2
                = 52.25 kg

            This is meaningful.












        Common examples of unit mismatch

            Height
                cm and inches

            Weight
                kg and lb

            Distance
                km and miles

            Temperature
                Celsius and Fahrenheit

            Money
                rupees and dollars

            Time
                hours and minutes









        Example with temperature

            Temperature values:
                25 C
                77 F

            If treated as 25 and 77 directly,
            analysis will be wrong.

            Convert 77 F to Celsius:
                (77 - 32) x 5/9
                = 25 C

            Now both values are same.









        Causes of unit mismatch

            Data collected from different countries
            Merging datasets from different sources
            Manual entry without standard rules
            Sensor devices using different measurement systems










        How to handle unit mismatch

            Identify units clearly
            Choose one standard unit
            Convert all values into that unit
            Document the conversion rule










        Example in business

            Sales data:
                100 USD
                8000 NPR

            These cannot be directly added.
            First convert to common currency.

        Important note
            Unit mismatch is dangerous because values may look normal,
            but the result becomes wrong silently.

            That is why data should always be checked carefully before analysis.









    Combined Example of All Issues

        Raw dataset

            Name      City         Weight      Marks
            Ram       KTM          55 kg       78
            Sita      Kathmandu    121 lb      82
            Hari      Pokhara      -           65
            Ram       KTM          55 kg       78

        Problems found

            Duplicate
                Ram appears twice

            Missing data
                Hari has missing weight

            Nonstandard data
                KTM and Kathmandu may need standard form

            Unit mismatch
                Weight uses kg and lb

        Cleaned idea

            Name      City         Weight(kg)    Marks
            Ram       Kathmandu    55            78
            Sita      Kathmandu    54.5          82
            Hari      Pokhara      Unknown       65

        Now data is more consistent and usable.

2.3 Data Cleaning and Standardization

    Data cleaning and standardization are important parts of data munging.
    They help convert messy raw data into a consistent and usable form.

    Simple meaning:
        Data cleaning means fixing errors in data.
        Data standardization means making data follow one common format.

    Why this is important
        Real-world data is often incomplete, inconsistent, or incorrect
        Different records may use different styles
        Analysis becomes unreliable if data is not cleaned
        Machine learning models perform better on clean and standardized data

    Main idea
        Before analysis, we must make sure the data is:
            Correct
            Complete enough
            Consistent
            Properly formatted









    Difference between cleaning and standardization
    -------------------------------------------------------
        Data cleaning
            Fixes problems in the data

            Example:
                Remove duplicate rows
                Correct wrong ages
                Handle missing marks

        Data standardization
            Makes the data follow one common rule or style

            Example:
                Convert all city names to one format
                Convert all dates to YYYY-MM-DD
                Convert all weights to kilograms






    Example of messy raw data
    ------------------------------
        Name        City         Weight      Date
        Ram         KTM          55 kg       01/02/24
        Sita        Kathmandu    121 lb      2024-02-01
        Hari        pokhara      -           Feb 1, 2024
        Ram         KTM          55 kg       01/02/24


    Problems
        Duplicate row
        Missing weight
        City names are inconsistent
        Weight units are mixed
        Date format is inconsistent

    After cleaning and standardization

        Name        City         Weight_kg     Date
        Ram         Kathmandu    55            2024-02-01
        Sita        Kathmandu    54.5          2024-02-01
        Hari        Pokhara      Unknown       2024-02-01

    Now the dataset is easier to understand and analyze.








    Main Tasks in Data Cleaning
    ------------------------------
        1. Remove duplicates
            Repeated records should be checked and removed when unnecessary

        2. Handle missing values
            Decide whether to remove, replace, or keep them as unknown

        3. Correct invalid values
            Example:
                Age = -5
                Marks = 500 out of 100

        4. Fix spelling and typing errors
            Example:
                Kathmadu should become Kathmandu

        5. Check data types
            Example:
                Age should be numeric
                Date should be date format

        6. Remove unwanted spaces and symbols
            Example:
                " Ram " should become "Ram"

        7. Correct inconsistent labels
            Example:
                Male, male, M should become one standard value

        8. Check for impossible values
            Example:
                Height = 0 cm
                Salary = negative amount










    Main Tasks in Data Standardization
    ------------------------------
        1. Standardize text format
            Make text use one style

            Example:
                kathmandu
                Kathmandu
                KATHMANDU

            Standardized form:
                Kathmandu

        2. Standardize units
            Convert all values into one common unit

            Example:
                55 kg
                121 lb

            Convert both into kilograms

        3. Standardize date format
            Use one date style throughout the dataset

            Example:
                01/02/24
                2024-02-01
                Feb 1, 2024

            Standardized form:
                2024-02-01

        4. Standardize category labels
            Example:
                M, Male, male
                F, Female, female

            Standardized form:
                Male
                Female

        5. Standardize number format
            Example:
                1,000
                1000
                1000.00

            Make format consistent as needed

        6. Standardize column names
            Example:
                Student Name
                student_name
                STUDENTNAME

            Standardized form:
                student_name

        7. Standardize missing value representation
            Example:
                -, NA, Null, blank

            Choose one form such as:
                NA
                or Unknown









    Step-by-Step Example
    ------------------------------
        Raw data

            Roll    Name        Gender     City         Weight      Marks
            1       Ram         M          KTM          55 kg       78
            2       Sita        Female     Kathmandu    121 lb      82
            3       Hari        male       pokhara      -           65
            1       Ram         M          KTM          55 kg       78



        Step 1: Remove duplicates

            Duplicate row for Roll 1 is removed

        Step 2: Handle missing values

            Hari's weight is missing
            We may keep it as Unknown
            or fill it later if suitable

        Step 3: Standardize gender

            M becomes Male
            male becomes Male
            Female stays Female

        Step 4: Standardize city

            KTM becomes Kathmandu
            pokhara becomes Pokhara

        Step 5: Standardize weight

            121 lb is converted to kg
            121 / 2.2 = 55 approximately

        Final cleaned data

            Roll    Name    Gender    City         Weight_kg    Marks
            1       Ram     Male      Kathmandu    55           78
            2       Sita    Female    Kathmandu    55           82
            3       Hari    Male      Pokhara      Unknown      65









    Common Cleaning Problems and Fixes
    ------------------------------
        Problem: Extra spaces

            Before:
                " Ram "
                " Sita"

            After:
                Ram
                Sita




        Problem: Mixed letter case

            Before:
                kathmandu
                KTM
                KATHMANDU

            After:
                Kathmandu





        Problem: Wrong data type

            Before:
                Age = "twenty"

            After:
                Age = 20
                if correct value is known





        Problem: Invalid numeric values

            Before:
                Marks = 105

            After:
                Correct if entry mistake
                or mark as invalid for review




        Problem: Different yes/no labels

            Before:
                Yes, Y, yes, 1
                No, N, no, 0

            After:
                Yes
                No








    Why Cleaning Must Be Done Carefully
    ---------------------------------------
        Cleaning is helpful,
        but careless cleaning can damage data.

        Example:
            If missing values are deleted without thinking,
            important records may be lost.

        Example:
            If all unusual values are removed,
            real but rare cases may disappear.

        So before cleaning, ask:
            Is this really an error?
            Is this value unusual but valid?
            Will removing it affect analysis?





    Example of unusual but valid data
    ------------------------------
        Age of students:
            18, 19, 20, 45

        At first, 45 may seem strange.
        But maybe that student is valid.
        So not every unusual value should be removed automatically.


    Data Cleaning in Maths Example

        Marks:
            45, 50, 60, 70, 700

        Problem:
            700 is likely data entry error

        Mean before cleaning:
            (45 + 50 + 60 + 70 + 700) / 5
            = 925 / 5
            = 185

        This is unrealistic.

        If corrected to 70:
            45, 50, 60, 70, 70

        Mean after cleaning:
            (45 + 50 + 60 + 70 + 70) / 5
            = 295 / 5
            = 59

        Lesson:
            Cleaning improves correctness of analysis.







    Data Standardization in Words Example
    ------------------------------
        Review words:
            Good
            good
            GOOD
            gOOD

        If counted directly, these may look different.

        After standardization to lowercase:
            good
            good
            good
            good

        Word frequency becomes correct.

        Another example

            Product names:
                Laptop
                laptop
                LAPTOP

            Standardization helps treat them as one category.














    Benefits of Data Cleaning and Standardization
    -------------------------------------------------
        Improves accuracy
        Reduces confusion
        Makes grouping easier
        Makes filtering easier
        Improves visualization
        Improves model performance
        Saves time in later analysis

    Risks if not done

        Wrong averages
        Wrong counts
        Wrong charts
        Poor machine learning results
        Misleading decisions











    Tools Used for Cleaning and Standardization
    --------------------------------------------
        Excel
            Sorting
            Filtering
            Find and replace
            Remove duplicates

        Python
            Pandas for cleaning and formatting

        R
            dplyr and tidyr packages

        SQL
            Query-based cleaning and transformation

        Data visualization tools
            Can help spot strange values












    Mini Practical Example
    ------------------------------
        Suppose we have sales data

            Product      Price       Date
            Rice         80         1/2/24
            rice         80         2024-02-01
            Sugar        -50        01-Feb-2024
            Oil          200        2024/02/01

        Cleaning steps

            Standardize product names
                Rice
                Rice
                Sugar
                Oil

            Fix invalid price
                Sugar price = -50 must be reviewed

            Standardize date
                2024-02-01 for all rows

        Cleaned version

            Product      Price      Date
            Rice         80         2024-02-01
            Rice         80         2024-02-01
            Sugar        Invalid    2024-02-01
            Oil          200        2024-02-01

        Then duplicates can be checked,
        and invalid values can be corrected.












    Best Practices
    ------------------------------
        Keep original raw data safe
        Make a cleaned copy instead of overwriting raw data immediately
        Document every cleaning step
        Use one standard format consistently
        Review suspicious values before deleting
        Check again after cleaning







    Final understanding
    ------------------------------
        Data cleaning removes errors and messy parts.
        Data standardization makes all records follow one common style.

        Both are necessary before good analysis.

        Main lesson:
            Raw data is rarely ready.
            Clean and standardize first,
            then analyze.










    Quick Revision

        Examples of standardization
            KTM becomes Kathmandu
            121 lb becomes 55 kg
            Feb 1, 2024 becomes 2024-02-01
            male and M become Male

        Main benefit
            Clean and consistent data leads to correct and meaningful analysis
